## 🚀 一键启动

### Step 1: 构建 common-lib

```bash
cd /Users/sanchezrick/Documents/Github/springboot-kafka-microservices
cd common-lib
mvn clean install -DskipTests -Dgpg.skip=true
cd ..
```

### Step 2: 构建所有微服务 JAR

```bash
for service in service-registry api-gateway order-service payment-service email-service product-service identity-service; do
  echo "Building $service..."
  cd $service && mvn clean package -DskipTests -Dgpg.skip=true && cd ..
done
```

### Step 3: Docker 启动所有容器

```bash
docker-compose up -d --build
```

### Step 4: 初始化数据库角色（⚠️ 首次启动必做！）

```bash
docker exec -i mysql-identity-service mysql -uroot -proot user_db -e "
INSERT INTO roles (name) VALUES ('Customer'), ('Employee'), ('Administrator');
"
```

### Step 5: 验证所有服务运行

```bash
docker-compose ps
open http://localhost:8761    # Eureka 控制台
```

## 🔐 API 使用指南

### 基础信息

```
API Gateway 地址：http://localhost:9191
Eureka 控制台：   http://localhost:8761
```

### 可用角色

| 角色名 | 说明 | 权限 |
|--------|------|------|
| CUSTOMER | 普通用户 | 下单、查看自己订单 |
| EMPLOYEE | 员工 | 管理商品、查看所有订单 |
| ADMINISTRATOR | 管理员 | 全部权限 |

---

### 1. 注册用户

```bash
# 注册普通用户
curl -X POST http://localhost:9191/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Test User",
    "email": "test@example.com",
    "password": "password123",
    "roles": ["CUSTOMER"]
  }'

# 返回：
# {"data":"User added to the system!","statusCode":201}

# 注册管理员
curl -X POST http://localhost:9191/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Admin User",
    "email": "admin@example.com",
    "password": "admin123",
    "roles": ["ADMINISTRATOR"]
  }'
```

### 2. 登录获取 Token

```bash
# ⚠️ 注意：
#   端点是 /token 不是 /login
#   username 用注册时的 name 字段，不是 email！

curl -X POST http://localhost:9191/api/v1/auth/token \
  -H "Content-Type: application/json" \
  -d '{
    "username": "Test User",
    "password": "password123"
  }'

# 返回：
# {"data":"eyJhbGciOiJIUzI1NiJ9.xxx...","statusCode":200}
#
# data 字段就是你的 JWT Token，下面要用到
```

### 3. 保存 Token（方便后续使用）

```bash
# 把 Token 存到变量里
TOKEN=$(curl -s -X POST http://localhost:9191/api/v1/auth/token \
  -H "Content-Type: application/json" \
  -d '{"username":"Test User","password":"password123"}' | python3 -c "import json,sys;print(json.load(sys.stdin)['data'])")

echo $TOKEN
```

### 4. 查看商品

```bash
# 查看所有商品（GET 不需要认证）
curl -s http://localhost:9191/api/v1/products | python3 -m json.tool
```

### 5. 创建商品（需要 EMPLOYEE 或 ADMINISTRATOR 角色）

```bash
# 先用管理员登录
ADMIN_TOKEN=$(curl -s -X POST http://localhost:9191/api/v1/auth/token \
  -H "Content-Type: application/json" \
  -d '{"username":"Admin User","password":"admin123"}' | python3 -c "import json,sys;print(json.load(sys.stdin)['data'])")

# 创建商品
curl -X POST http://localhost:9191/api/v1/products \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -d '{
    "name": "iPhone 15",
    "description": "Latest iPhone",
    "price": 999.99,
    "categoryId": 1
  }'
```

### 6. 下单（需要 CUSTOMER 角色 + Token）

```bash
curl -X POST http://localhost:9191/api/v1/order \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "userId": "user-123",
    "orderItems": [
      {
        "productId": "prod-001",
        "variantId": 1,
        "quantity": 2,
        "price": 29.99
      }
    ]
  }'
```

### 7. 验证 Token

```bash
curl -s "http://localhost:9191/api/v1/auth/validate?token=$TOKEN" | python3 -m json.tool
```

### 8. 获取当前用户信息

```bash
curl -s http://localhost:9191/api/v1/auth/me \
  -H "Authorization: Bearer $TOKEN" | python3 -m json.tool
```

## ⚠️ 常见问题

### Q: 注册报 "Invalid role name: ROLE_USER"
**A:** 角色名不能带 `ROLE_` 前缀。正确的值是 `CUSTOMER`、`EMPLOYEE`、`ADMINISTRATOR`。

### Q: 注册报 "Role not found"
**A:** 首次启动没有初始化角色表。运行：
```bash
docker exec -i mysql-identity-service mysql -uroot -proot user_db -e "
INSERT INTO roles (name) VALUES ('Customer'), ('Employee'), ('Administrator');"
```

### Q: 登录报 "Bad credentials"
**A:** username 用的是注册时的 **name** 字段，不是 email。比如注册时 `name: "Test User"`，登录时 `username: "Test User"`。

### Q: 登录端点返回 404
**A:** 登录端点是 `/api/v1/auth/token`，不是 `/api/v1/auth/login`。

### Q: API Gateway 端口是多少？
**A:** **9191**，不是 8080。

### Q: curl 请求返回 503 Service Unavailable
**A:** 服务还没注册到 Eureka。Docker 起来后需要等 30-60 秒让所有服务完成注册。可以在 http://localhost:8761 查看。

### Q: 服务启动失败怎么查日志？
```bash
docker-compose logs -f <service-name>
# 例如：docker-compose logs -f identity-service
```
