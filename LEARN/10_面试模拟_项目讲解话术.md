## 📌 面试开场：30秒介绍你的项目（背下来！）

### 中文版：

> "我实践了一个基于 Spring Boot 3 的微服务电商系统。
> 系统拆分成7个独立服务：商品、订单、支付、邮件、身份认证、
> API网关和服务注册中心。
> 服务间的同步通信用 OpenFeign，异步通信用 Apache Kafka。
> 用户认证通过 JWT 实现，在 API Gateway 层统一验证。
> 每个服务有独立的 MySQL 数据库，用 Redis 做缓存加速。
> 整个系统通过 Docker Compose 一键部署。"

### English Version（JD 面试用英文！）：

> "I built a microservices-based e-commerce system using Spring Boot 3.
> The system consists of 7 independent services: product, order, payment,
> email notification, identity, API gateway, and service registry.
> Services communicate synchronously via OpenFeign and asynchronously
> through Apache Kafka. Authentication is handled by JWT tokens,
> validated at the API Gateway layer.
> Each service has its own MySQL database, with Redis for caching.
> The entire system is containerized and orchestrated with Docker Compose."

## 📌 面试官追问模拟（一问一答）

> 以下按面试官最可能问的**顺序**排列。
> 每个问题先给出中文答案思路，再给英文回答要点。

---

### Round 1: 项目整体（100% 会问）

---

**Q1: Walk me through the architecture of your project.**

> 🗣️ 答题思路：画一个流程图式的回答
>
> "The system has three layers:
>
> First, the **API Gateway** - it's the single entry point.
> All client requests come through port 9191.
> It does JWT validation and routes requests to the correct service.
>
> Second, **business services** - Order, Product, Payment, Identity, Email.
> Each is a standalone Spring Boot application with its own database.
> They register themselves with **Eureka** for service discovery.
>
> Third, **infrastructure** - MySQL databases for persistence,
> Redis for caching, and Kafka for asynchronous messaging.
>
> For example, when a user places an order:
> API Gateway validates the JWT → forwards to Order Service →
> Order Service saves the order and publishes a Kafka event →
> Payment Service and Email Service consume the event independently."

---

**Q2: Why did you choose microservices over a monolith?**

> 🗣️ "For an e-commerce platform, different modules have different
> scaling needs. During high traffic like sales events,
> the order service needs more instances while the email service doesn't.
> Microservices let us scale each component independently.
> Also, different teams can work on different services simultaneously
> without code conflicts."
>
> ⚠️ **加分回答：** "However, for a small team or simple project,
> I'd choose a monolith first because microservices add complexity
> in distributed transactions, network reliability, and operations."

---

### Round 2: Spring Boot 核心（90% 会问）

---

**Q3: What is dependency injection? Why do we need it?**

> 🗣️ "Instead of creating objects ourselves with `new`,
> we declare what we need and Spring creates and provides them.
>
> For example, my PaymentServiceImpl needs a PaymentRepository.
> I declare it as a final field, and Spring injects it through
> the constructor. This makes the code loosely coupled,
> testable - I can pass a mock in unit tests without Spring -
> and flexible - I can swap implementations without changing code."

---

**Q4: What does @SpringBootApplication do?**

> 🗣️ "It's a combination of three annotations:
> @Configuration marks it as a config class,
> @EnableAutoConfiguration tells Spring Boot to auto-configure
> things like database connections based on dependencies in pom.xml,
> and @ComponentScan scans the package for @Service, @Repository,
> @Controller and registers them as beans."

---

**Q5: How does auto-configuration work?**

> 🗣️ "Spring Boot checks which dependencies are on the classpath.
> For example, when it sees mysql-connector-java, it auto-configures
> a DataSource bean. When it sees spring-boot-starter-web,
> it auto-configures an embedded Tomcat.
> Each auto-configuration class uses @ConditionalOnClass
> to only activate when the relevant library is present."

---

### Round 3: Database & JPA（80% 会问）

---

**Q6: How do you interact with the database?**

> 🗣️ "I use Spring Data JPA. Each database table has a corresponding
> Entity class annotated with @Entity. For data access,
> I define interfaces extending JpaRepository.
> Spring automatically generates SQL from method names -
> for example, `findByOrderId` becomes
> `SELECT * FROM payments WHERE order_id = ?`.
> For complex queries, I can use @Query with JPQL."

---

**Q7: What is the N+1 problem?**

> 🗣️ "When loading entities with a collection relationship like
> @OneToMany, fetching N parent entities triggers N additional
> queries for each parent's children. So 100 orders with items
> causes 1 + 100 = 101 queries instead of 1 or 2.
> I solve this with JOIN FETCH in JPQL queries,
> or @EntityGraph to specify eager loading for specific queries."

---

### Round 4: Kafka（70% 会问，JD 电商系统核心）

---

**Q8: Why use Kafka instead of direct HTTP calls?**

> 🗣️ "Three reasons:
> **Decoupling** - Order Service doesn't need to know about
> Payment or Email Services. It just publishes events.
> **Async processing** - Users don't wait for email sending
> or payment processing. The response is instant.
> **Peak shaving** - During sales events, Kafka buffers messages
> so downstream services aren't overwhelmed."

---

**Q9: How do you handle message ordering and duplication?**

> 🗣️ "Kafka guarantees ordering within a partition.
> For operations that need ordering, I use the order ID as the
> message key, so all events for the same order go to the same
> partition.
>
> For duplication, I implement idempotency at the consumer side.
> Before processing a payment, I check if a payment already exists
> for that order ID. If it does, I skip processing."

---

### Round 5: Security（60% 会问）

---

**Q10: How does authentication work in your system?**

> 🗣️ "Users authenticate through the Identity Service,
> which validates credentials and returns a JWT token.
> For subsequent requests, the token is sent in the
> Authorization header.
> The API Gateway has a global filter that intercepts every request,
> extracts and validates the JWT.
> If valid, the request is forwarded to the target service.
> If invalid or expired, it returns 401 Unauthorized."

---

**Q11: Why JWT over Session?**

> 🗣️ "In a microservices architecture, sessions require
> shared storage like Redis across all services.
> JWT is self-contained - any service can validate the token
> independently by checking the signature.
> This is more scalable and stateless,
> which aligns well with REST principles."

---

### Round 6: Docker（50% 会问）

---

**Q12: How do you deploy your services?**

> 🗣️ "Each service has a Dockerfile based on eclipse-temurin:17-jre.
> I compile the JAR with Maven, then Docker copies it into the image.
> Docker Compose orchestrates all 14 containers -
> 7 services plus MySQL, Redis, Kafka, and Zookeeper.
> Services use Docker's internal DNS to communicate by service name
> rather than IP addresses."

---

## 📌 高级追问（如果面试官挖深）

---

**Q: How would you handle a service being down?**

> 🗣️ "I'd implement circuit breakers using Resilience4j.
> If Payment Service is down, the circuit opens and requests
> fail fast instead of hanging.
> For async communication through Kafka,
> the messages are persisted in the topic.
> When Payment Service recovers, it consumes the backlog."

---

**Q: How would you handle distributed transactions?**

> 🗣️ "I use the Saga pattern with choreography.
> Each service publishes events after completing its step.
> If payment fails, a compensating event is published
> to cancel the order. This achieves eventual consistency
> without needing distributed locks like 2PC."

---

**Q: What would you improve in this project?**

> 🗣️ "Three things:
> First, add distributed tracing with Zipkin to track requests
> across services.
> Second, implement circuit breakers for resilience.
> Third, add comprehensive integration tests and CI/CD pipeline
> with GitHub Actions for automated deployment."

## 📌 面试 Checklist

### 面试前一天：
- [ ] 能用英语 30 秒介绍项目
- [ ] 能画出架构图（纸上画）
- [ ] 每个知识点有 1 个项目代码例子
- [ ] 准备了 2-3 个问面试官的问题

### 面试中的技巧：
- ✅ 不知道就说 "I'm not sure about the details, but my understanding is..."
- ✅ 每个回答连接到**你的项目**："In my project, I used this in..."
- ✅ 说完理论加一句实践："For example, in my order service..."
- ❌ 不要背诵定义，用自己的话解释
- ❌ 不要说 "I just followed a tutorial"，说 "I practiced building..."
